# WS-5 — The Onyx Line for Jobs (realtime + contact-safe masking + deep-link convergence)

**Program:** The Onyx Line (`docs/superpowers/plans/2026-06-26-the-onyx-line.md`)
**Branch:** `v3/typography-reading-foundation` (kept-as-is, not pushed)
**Status:** Planned. Grounded in live source (read 2026-06-26).
**Posture:** Enhancement of an EXISTING sanctioned channel (candidate↔employer DM is live, on-platform policy explicit). Migration committed-NOT-applied. Money untouched.

## North Star

Bring the jobs candidate↔employer thread up to the Onyx Line standard: (1) close the **contact-masking gap** (medium-severity contact details currently persist verbatim), (2) make the thread **live** (realtime), (3) adopt the **shared messaging-thread engine** for one consistent surface, and (4) **deep-link** jobs threads from the unified inbox instead of dumping users at `/jobs`.

**Key difference from WS-4 (marketplace):** jobs is **NOT identity-minimized** — in a hiring conversation the candidate and employer legitimately see each other's names. So typing presence stays ON; no "Buyer"-style anonymization. The invariant that DOES apply is **no contact-detail leak** (anti-disintermediation: "Keep all communication, contact details, and payment arrangements inside Henry & Co.").
